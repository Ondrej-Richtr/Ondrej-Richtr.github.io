3D Game using Raylib

Year: 2023, Programmed in:  Zig

A 3D game made using Raylib with simple movement and collision mechanics.

My first attempt at a 3D game. I used simple game programming library Raylib for rendering and I/O. Project is (again) entirely in Zig, seems like a good combination with Raylib (except it fails to compile for web).

The “game” is just a playground of stuff I was testing out, you can build and delete walls or platforms, fly or jump around and there’s also a ‘mirror’ that reflects what you see in the game. I also tried to implement collision mechanics, they are getting close to being finished, currently collisions work only with walls, platforms and the ground.

You can download portable version for Windows x86_64 here (ZIP).

Controls

  • Mouse to look.
  • W, S, A, D to move.
  • G to teleport to ground level (useful if you clip through the ground).
  • 1 to switch into ‘Ray’ mode - Left Mouse to shoot a ray, Right Mouse to delete all rays.
  • 2 to switch into ‘Build’ mode - Left/Right Arrows to switch between wall building and platform building, Up/Down Arrows to adjust build height. Hold Left Mouse to build a structure, Right Mouse to delete new structure, Enter to confirm new structure.
  • 3 to switch into ‘Delete’ mode - Left Mouse to delete a wall/platform you aim at.
  • Shift + 1 to switch into ‘Game view’ - Standard first person movement with gravity and collisions, use Space key to jump.
  • Shift + 2 to switch into ‘Free view’ - First person free view without gravity and collisions.
  • Esc to exit the game.

I had a break from this project because of my studies, but I returned to it and work on it whenever I can. Currently I’m working on proper player collision model (something better than a sphere at player’s feet) and moving platforms.

The long term goal is probably going to be a FPS game with good movement/collision mechanics. I also want to implement my own 3D mesh handling as Raylib is pretty limiting in this regard.
